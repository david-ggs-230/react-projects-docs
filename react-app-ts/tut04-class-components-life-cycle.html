4.4. Class Components - Life Cycle

Each React component has three distinct stages:

  • Mounting − Mounting represents the rendering of the React component in the given DOM node.

    • constructor()

      Invoked during the initial construction phase of the React component. Used to set initial state and properties of the component.

    • getDerivedStateFromProps()

      Invoked during both initial and update phase and just before the render() method. It is mostly used in animation context where the various state of the component is needed to do smooth animation. The signature of the API is: static getDerivedStateFromProps(props, state)

      • props − current properties of the component

      • state − Current state of the component

    • render()

      It renders the component in the virtual DOM instance. This is specified as mounting of the component in the DOM tree.

    • componentDidMount()

      Invoked after the initial mounting of the component in the DOM tree.

  • Updating − Updating represents the re-rendering of the React component in the given DOM node during state changes / updates.

    • shouldComponentUpdate()

      Invoked during the update phase. Used to specify whether the component should update or not. The signature of the API is: shouldComponentUpdate(nextProps, nextState)

      • nextProps − Upcoming properties of the component

      • nextState − Upcoming state of the component

    • getDerivedStateFromProps()

      Invoked during both initial and update phase and just before the render() method. It is mostly used in animation context where the various state of the component is needed to do smooth animation. The signature of the API is: static getDerivedStateFromProps(props, state)

      • props − current properties of the component

      • state − Current state of the component

    • render()

      It renders the component in the virtual DOM instance.

    • getSnapshotBeforeUpdate()

      Invoked just before the rendered content is commited to DOM tree. The data returned by this method will be passed to ComponentDidUpdate() method. The signature of the API is: getSnapshotBeforeUpdate(prevProps, prevState)

      • prevProps − Previous properties of the component.

      • prevState − Previous state of the component.

    • componentDidUpdate()

      Invoked during the update phase. The signature of the API is: componentDidUpdate(prevProps, prevState, snapshot)

      • prevProps − Previous properties of the component.

      • prevState − Previous state of the component.

      • snapshot − Current rendered content.

  • Unmounting − Unmounting represents the removal of the React component.

    • componentWillUnmount()

      Invoked after the component is unmounted from the DOM. This is the good place to clean up the object.

4.4.1. Create a React Project Structure

4.4.1.1. Create a React Project

  • Create a ReactJS Project

    yarn create vite tut04-class-components-life-cycle --template react-ts

  • Move inside the ReactJS project folder <tut04-class-components-life-cycle>

    cd tut04-class-components-life-cycle

  • Install the dependencies

    yarn install

4.4.2. Create Class Components Showing Life Cycles

Life Cycle Stages:

  • Mounting

    • constructor()

    • getDerivedStateFromProps()

    • render()

    • componentDidMount()

  • Updating

    • shouldComponentUpdate()

    • getDerivedStateFromProps()

    • render()

    • getSnapshotBeforeUpdate()

    • componentDidUpdate()

  • Unmounting

    • componentWillUnmount()

4.4.2.1. Class Components with Life Cycle Methods

Define a class component with life cycle methods.

Listing 4.33 src/ComponentLifeCycle.tsx
  1import React from "react";
  2interface ComponentLifeCycleState {
  3  count: number;
  4  rerender: boolean;
  5}
  6
  7class ComponentLifeCycle extends React.Component<
  8  object,
  9  ComponentLifeCycleState
 10> {
 11  static logMessages: string[] = [];
 12  constructor(props: object) {
 13    super(props);
 14    this.state = {
 15      count: 0,
 16      rerender: true,
 17    };
 18    if (!ComponentLifeCycle.logMessages) ComponentLifeCycle.logMessages = [];
 19    ComponentLifeCycle.logMessages.push(
 20      "constructor(), count: " + this.state.count,
 21    );
 22  }
 23
 24  static getDerivedStateFromProps(
 25    props: object,
 26    state: ComponentLifeCycleState,
 27  ): Partial<ComponentLifeCycleState> | null {
 28    ComponentLifeCycle.logMessages.push(
 29      "getDerivedStateFromProps(), count: " + state.count,
 30    );
 31    return null;
 32  }
 33
 34  componentDidMount() {
 35    ComponentLifeCycle.logMessages.push(
 36      "componentDidMount(), count: " + this.state.count,
 37    );
 38  }
 39
 40  shouldComponentUpdate(nextProps: object, nextState: ComponentLifeCycleState) {
 41    ComponentLifeCycle.logMessages.push(
 42      "shouldComponentUpdate(), count: " + nextState.count,
 43    );
 44    return true;
 45  }
 46
 47  getSnapshotBeforeUpdate(
 48    nextProps: object,
 49    nextState: ComponentLifeCycleState,
 50  ) {
 51    ComponentLifeCycle.logMessages.push(
 52      "getSnapshotBeforeUpdate(), count: " + this.state.count,
 53    );
 54    return nextState.count;
 55  }
 56
 57  componentDidUpdate() {
 58    ComponentLifeCycle.logMessages.push(
 59      "componentDidUpdate(), count: " + this.state.count,
 60    );
 61  }
 62
 63  componentWillUnmount() {
 64    ComponentLifeCycle.logMessages.push(
 65      "componentWillUnmount(), count: " + this.state.count,
 66    );
 67  }
 68
 69  handleIncrement = () => {
 70    this.setState({ count: this.state.count + 1 });
 71    ComponentLifeCycle.logMessages.push(
 72      "handleIncrement (), count: " + this.state.count,
 73    );
 74  };
 75
 76  render() {
 77    ComponentLifeCycle.logMessages.push("render(), count: " + this.state.count);
 78    return (
 79      <>
 80        <div className="list-container">
 81          <h2>Class Component Life Cycle</h2>
 82          <p>
 83            Count: {this.state.count}{" "}
 84            <button onClick={this.handleIncrement}>Increment</button>
 85          </p>
 86          <h4>Log Messages:</h4>
 87          <ol>
 88            {ComponentLifeCycle.logMessages.map((message, index) => (
 89              <li key={index} className="list-item" style={{ margin: "1px" }}>
 90                <div>
 91                  {index + 1}. {message}
 92                </div>
 93              </li>
 94            ))}
 95          </ol>
 96        </div>
 97      </>
 98    );
 99  }
100}
101
102export default ComponentLifeCycle;

4.4.4. Hosting the React App on GitHub Pages

4.4.4.1. Build the App

  • Configure the ts build options:

    • open tsconfig.app.json file

    • set “noUnusedParameters”: false, as some parameters are place-holders required for life cycle functions

      /* Linting */
"strict": true,
"noUnusedLocals": true,
"noUnusedParameters": false,
"noFallthroughCasesInSwitch": true,
"noUncheckedSideEffectImports": true

  • Configure the build base url:

    • open vite.config.js file

    • set base to /react-projects/react-projects-with-typescript/tut04-class-components-life-cycle/

      export default defineConfig({
    plugins: [react()],
    base: "/react-projects/react-projects-with-typescript/tut04-class-components-life-cycle/",
})

  • Build the app

    yarn run build
